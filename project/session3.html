Scope Management

Session 3 - Defining and Controlling Project Boundaries

Department of Industrial Management | University of Kelaniya

What is Scope Management?

Scope management involves defining and controlling what is included and excluded in a project. It ensures the project includes all the work required—and only the work required—to complete the project successfully.

Why is Scope Management Important?

  • Prevents scope creep (uncontrolled changes to project scope)
  • Ensures project delivers what was promised
  • Helps manage stakeholder expectations
  • Provides basis for scheduling and budgeting
  • Reduces risk of project failure
1. What is the primary purpose of scope management?



Correct Answer: To define and control what's included in the project
Explanation: Scope management focuses on ensuring the project includes all necessary work (and only that work) to meet objectives.

Scope Management Processes

According to PMBOK, scope management includes six key processes:

Process Description Key Output
Plan Scope Management Create a scope management plan that documents how project scope will be defined, validated, and controlled Scope Management Plan
Collect Requirements Determine, document, and manage stakeholder needs and requirements Requirements Documentation
Define Scope Develop detailed description of the project and product Project Scope Statement
Create WBS Subdivide project deliverables into smaller, manageable components Work Breakdown Structure (WBS)
Validate Scope Formalize acceptance of completed project deliverables Accepted Deliverables
Control Scope Monitor project and product scope and manage changes Change Requests

Case Study: E-Commerce Website Development

A company is developing a new e-commerce website. Their scope management process includes:

  • Plan Scope: Document how scope will be managed
  • Collect Requirements: Interview stakeholders to identify needed features
  • Define Scope: Create a detailed scope statement including all features
  • Create WBS: Break down work into manageable tasks (design, development, testing)
  • Validate Scope: Get client sign-off on completed features
  • Control Scope: Use change requests for any new feature additions

Scope Statement

A scope statement is a document that clearly defines the project boundaries, deliverables, and constraints.

Key Components of a Scope Statement

  • Project Objectives: What the project aims to achieve
  • Deliverables: Tangible outputs of the project
  • Acceptance Criteria: Conditions that must be met for deliverables to be accepted
  • Constraints: Limitations (time, budget, resources)
  • Assumptions: Factors considered true for planning purposes
  • Exclusions: What is specifically not included

Example Scope Statement (Bank E-Commerce Application)

  • Develop an electronic commerce strategy identifying processes, products and services
  • Create an application system supporting all identified processes
  • Integrate with the bank's existing ERP system
  • Exclusion: Mobile app development is not included in this phase
2. Which of these would NOT typically be included in a project scope statement?



Correct Answer: Detailed work breakdown structure
Explanation: While the WBS is related to scope, it's a separate document that follows the scope statement.

Work Breakdown Structure (WBS)

The WBS is a hierarchical decomposition of the total scope of work to be carried out by the project team.

Key Characteristics of WBS

  • 100% Rule: Includes ALL work required for the project
  • Deliverable-oriented: Focuses on outputs, not activities
  • Hierarchical structure: From general to specific
  • Mutually exclusive elements: No overlap between components
  • Defines project scope baseline

WBS Levels

Level Name Description
1 Project The overall project
2 Deliverables Major project components
3 Sub-deliverables Supporting deliverables
4+ Work Packages Individual project activities (lowest level)

Example WBS (House Construction)

1.0 House Construction
  1.1 Foundation
    1.1.1 Excavation
    1.1.2 Concrete Pouring
  1.2 Framing
    1.2.1 Wall Construction
    1.2.2 Roof Construction
  1.3 Interior
    1.3.1 Electrical
    1.3.2 Plumbing
3. Which of the following is NOT a characteristic of a proper WBS?



Correct Answer: Includes overlapping elements between components
Explanation: A proper WBS has mutually exclusive elements with no overlap between components.

WBS Dictionary

The WBS dictionary provides detailed information about each component in the WBS.

Contents of WBS Dictionary

  • Code of account identifier
  • Description of work
  • Assumptions and constraints
  • Responsible organization
  • Schedule milestones
  • Resources required
  • Cost estimates
  • Quality requirements
  • Acceptance criteria

Example WBS Dictionary Entry

Field Example Content
WBS Code 1.3.1
Name Electrical Work
Description Install all electrical wiring, outlets, and fixtures
Responsible ABC Electrical Contractors
Milestones Complete rough-in by March 15, final inspection by April 1
Budget $25,000

Scope Verification and Control

Ensuring the project stays on track by managing scope changes.

Scope Verification

  • Formal acceptance of completed project deliverables
  • Occurs at the end of each phase or major deliverable
  • Involves inspections, reviews, or testing
  • Results in accepted deliverables or change requests

Scope Control

  • Process of monitoring project scope and managing changes
  • Key tool: Change Control System (formal process for evaluating changes)
  • Prevents scope creep (unauthorized changes)
  • Documents all change requests and decisions

Scope Change Request Process

  1. Submit formal change request
  2. Assess impact on scope, schedule, cost
  3. Review by Change Control Board (CCB)
  4. Approve/reject decision
  5. Update project documents if approved
  6. Communicate decision to stakeholders
  7. Implement approved changes
4. What is the primary purpose of scope control?



Correct Answer: To monitor scope and manage changes to prevent scope creep
Explanation: Scope control focuses on ensuring the project stays within its defined boundaries by properly managing any changes.

Key Takeaways

  • Scope management defines and controls what is included in the project
  • The scope statement clearly documents project boundaries and deliverables
  • The WBS breaks down work into manageable components following the 100% rule
  • The WBS dictionary provides detailed information about each WBS component
  • Scope verification ensures deliverables meet requirements
  • Scope control manages changes to prevent scope creep
  • Proper scope management is critical to project success
