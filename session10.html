Session 1-2: Software Quality Fundamentals

1. Compare and contrast Software Quality Assurance (SQA) and Software Testing (QC)

SQA (Quality Assurance):

  • Process-oriented approach
  • Focuses on defect prevention
  • Activities throughout SDLC
  • Establishes processes and standards

QC (Quality Control/Testing):

  • Product-oriented approach
  • Focuses on defect identification
  • Activities primarily during testing phase
  • Executes tests to find deviations

2. Explain the terms failure, error, and fault with examples

Fault (Defect/Bug): A static defect in the code (e.g., incorrect interest calculation formula)

Error: Incorrect internal state due to fault (e.g., wrong interest value calculated)

Failure: Observable incorrect behavior (e.g., customer receives wrong interest amount)

Session 3: Earning Software Quality

3. List 5 key benefits of Software Quality Assurance

  1. Reduces long-term development costs
  2. Improves customer satisfaction
  3. Enhances product reliability
  4. Reduces maintenance efforts
  5. Improves team productivity

4. Explain two main features of quality code with characteristics

Readability:

  • Consistent indentation
  • Appropriate comments

Maintainability:

  • Modular design
  • Low coupling
  • High cohesion

5. Explain four black box testing techniques

  1. Equivalence Partitioning: Dividing input data into valid/invalid classes
  2. Boundary Value Analysis: Testing at the edges of input ranges
  3. Decision Table Testing: Testing combinations of inputs and conditions
  4. State Transition Testing: Testing behavior between different states

7. Explain three code coverage techniques in white box testing

  1. Statement Coverage: Executes all statements at least once
  2. Branch Coverage: Executes all decision outcomes (true/false)
  3. Path Coverage: Executes all possible paths through the code

Session 7-8: Testing Principles & Automation

8. Compare verification and validation in software testing

Verification: "Are we building the product right?" (Process-focused)

  • Reviews, walkthroughs, inspections
  • Checks against specifications

Validation: "Are we building the right product?" (Product-focused)

  • Actual testing activities
  • Checks against user needs

9. List three reasons to use test automation

  1. To execute repetitive regression tests efficiently
  2. To perform tests that are difficult to do manually (e.g., performance testing)
  3. To enable continuous testing in CI/CD pipelines

Practical Coding Questions

12. Write pseudocode for Bubble Sort algorithm

procedure bubbleSort(A : list of sortable items)
    n = length(A)
    repeat
        swapped = false
        for i = 1 to n-1 inclusive do
            if A[i-1] > A[i] then
                swap(A[i-1], A[i])
                swapped = true
            end if
        end for
        n = n - 1
    until not swapped
end procedure

13. Calculate Cyclomatic Complexity for the Bubble Sort code

Formula: V(G) = E - N + 2P

Where E = edges, N = nodes, P = connected components

For the bubble sort:

  • Decision points: 1 (if condition)
  • V(G) = 1 (if) + 1 (repeat) = 2
  • Cyclomatic Complexity = 2
