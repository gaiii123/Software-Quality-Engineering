Playwright Implementation at GTN

Why GTN Chose Playwright:
  • Cross-browser testing (Chromium, WebKit, Firefox)
  • Auto-waiting and web-first assertions
  • Single API for JavaScript/TypeScript
  • Parallel test execution capability
  • Excellent debugging tools
Challenges Faced:
  • Learning curve for non-JavaScript developers
  • Mobile testing limitations
  • Newer tool with smaller community than Selenium

3. Defining QA Boundaries at GTN

In Scope for QA Team

  • Functional Testing: All business requirements validation
  • UI Testing: Cross-browser and responsive testing
  • API Testing: Contract and integration testing
  • Regression Testing: Automated test suite execution
  • Performance Testing: Basic load testing for critical flows

Out of Scope for QA Team

  • Unit Testing: Owned by development team
  • Infrastructure Testing: Handled by DevOps
  • Security Penetration Testing: Conducted by specialized team
  • UX Design Validation: Owned by product designers
  • Content Accuracy: Verified by content team
