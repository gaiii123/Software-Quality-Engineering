Introduction to Software Quality Engineering

Session Overview

  • Introduction to the Module
  • Discuss evaluation criteria
  • Understand need of Software Quality Engineering
  • Identify different roles in Software Quality Engineering

Learning Outcomes of the Module

  • Identify the role and responsibilities of quality engineering
  • Demonstrate a sound understanding of software quality assurance process
  • Apply different software validation and verification techniques
  • Conduct software inspections, reviews and assess error proneness and testability
  • Design and implement comprehensive test plans
  • Apply various software testing techniques in an effective and efficient manner
  • Determine test coverage and yield according to different criteria
  • Apply different software testing tools

Course Content

Software quality engineering, software quality assurance process, software validation and verification, software inspections and reviews, software testing principles, test design techniques, different testing types, defect and defect reporting, QA documentation and QA metrics, software error proneness and testability, specification-based testing, code-based testing, alpha, beta and acceptance testing, test prioritisation, coverage criteria, software process and product quality, software quality standards, software testing tools (UI automation, load testing, performance testing, code quality analysis, security testing etc.), statistical techniques to evaluate the defect density and reliability.

Readings

  • Jeff Tijan (2005), Software Quality Engineering: Testing, Quality Assurance, and Quantifiable Improvement 1st Edition, Wiley-IEEE Computer Society Press

  • Arnon Axelrod (2018), Complete Guide to Test Automation: Techniques, Practices, and Patterns for Building and Maintaining Effective Software Projects 1st ed. Edition, Apress
  • Geradus Blokdyk (2020), Acceptance Test Driven Development A Complete Guide - Kindle Edition 5STARCooks
